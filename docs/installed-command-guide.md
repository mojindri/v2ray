# Installed Commands

This guide is for Linux VPS installs from GitHub Releases.
For the Black UI companion panel, see [black-ui.md](black-ui.md).

## Where Things Are

```text
/usr/local/bin/blackwire       binary
/etc/blackwire/config.json     main config
/etc/blackwire/client-info.txt generated client connection hints
/var/lib/blackwire             service working directory
/etc/systemd/system/blackwire.service systemd unit
```

The binary should be available as `blackwire` without typing the full path:

```sh
which blackwire
blackwire version
```

## Start / Stop

```sh
sudo systemctl status blackwire --no-pager
sudo systemctl start blackwire
sudo systemctl stop blackwire
sudo systemctl restart blackwire
sudo systemctl reload blackwire
```

Enable or disable start-on-boot:

```sh
sudo systemctl enable blackwire
sudo systemctl disable blackwire
```

For `SETUP=domain`, nginx is also part of the runtime:

```sh
sudo systemctl status nginx --no-pager
sudo systemctl restart nginx
sudo nginx -t
```

## Logs

```sh
sudo journalctl -u blackwire -f
sudo journalctl -u blackwire -n 100 --no-pager
```

For nginx domain setup:

```sh
sudo journalctl -u nginx -n 100 --no-pager
sudo tail -n 100 /var/log/nginx/error.log
```

## Change Config

Edit:

```sh
sudo nano /etc/blackwire/config.json
```

Validate before applying:

```sh
blackwire test -c /etc/blackwire/config.json
```

Apply:

```sh
sudo systemctl reload blackwire
```

If reload is not enough for the kind of change you made:

```sh
sudo systemctl restart blackwire
```

## Client Info

Generated VPS setup writes connection details here:

```sh
sudo cat /etc/blackwire/client-info.txt
```

This is where to find UUID/password, domain, port, path, REALITY public key,
short ID, and similar values generated by the installer.

## Check Access

For `SETUP=domain`, Blackwire listens on localhost and nginx owns public HTTPS:

```sh
sudo ss -ltnp | grep -E 'blackwire|nginx'
sudo systemctl status nginx --no-pager
sudo nginx -t
curl -I https://your-domain.example/
```

Expected shape:

```text
nginx:    0.0.0.0:80 and 0.0.0.0:443
blackwire: 127.0.0.1:10080
```

For direct/reality setup:

```sh
sudo ss -ltnp | grep blackwire
```

## Upgrade

Release installer:

```sh
curl -fsSL https://raw.githubusercontent.com/mojindri/Blackwire/v0.1.0-rc.4/scripts/install.sh \
  | VERSION=v0.1.0-rc.4 ACTION=upgrade bash
```

Downloaded Debian package:

```sh
sudo apt install ./blackwire_<version>_amd64.deb
```

## Uninstall

Keep config and state:

```sh
curl -fsSL https://raw.githubusercontent.com/mojindri/Blackwire/v0.1.0-rc.4/scripts/install.sh \
  | ACTION=uninstall bash
```

Remove config and state too:

```sh
curl -fsSL https://raw.githubusercontent.com/mojindri/Blackwire/v0.1.0-rc.4/scripts/install.sh \
  | ACTION=uninstall REMOVE_CONFIG=1 bash
```

Debian package:

```sh
sudo apt remove blackwire
sudo apt purge blackwire
```
